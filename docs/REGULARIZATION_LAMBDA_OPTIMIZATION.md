## 🎯 최적 λ 값 도출

### 방법론: Penalty의 역할

Regularization penalty는:
```
Total Score = MSE + λ × |α - 1.0|
```

**λ의 의미**:
- MSE를 얼마나 희생하면서 α=1.0에 가까운 값을 선호할 것인가?
- λ가 클수록 α=1.0 근처만 선택
- λ가 작을수록 MSE만 보고 선택

### 현재 데이터 기반 분석

**MSE 스케일**: ~1.02 (RMSE ~1.01)

**성능 개선/악화 규모**:
- **성능 개선** 케이스: 평균 +1.13% (α 편차: 0.44)
- **성능 악화** 케이스: 평균 -3.17% (α 편차: 2.84)

**핵심 인사이트**:
```
α 편차 1.0 이상 → 98.6% 확률로 성능 악화 (-3.17%)
α 편차 0.5 이하 → 대부분 성능 개선 또는 유지
```

### λ 값 시뮬레이션 결과

**시나리오**: MSE=1.0 기준, α=0.7이 2% 개선, α=1.3이 1% 개선

```
λ        α=0.7 선택?    α=1.0 선택?
─────────────────────────────────
0.01     ✅ (너무 공격적)
0.05     ✅ (공격적)
0.10     ❌ (보수적, α=1.0 선택)
```

**해석**:
- λ=0.01: α=0.7 선택 (2% MSE 개선 얻기 위해 0.3 편차 허용)
- λ=0.05: α=0.7 선택 (하지만 penalty 증가)
- λ=0.10: α=1.0 선택 (2% MSE 개선을 포기하고 안정성 선택)

### 실제 케이스 분석

**대표 사례 (FOLD 2, PCC, K=20)**:
```
Alpha    MSE       Penalty(λ=0.01)   Total      선택?
──────────────────────────────────────────────────────
1.0      0.7188    0.0000            0.7188
1.5      0.7067    0.0050            0.7117
2.0      0.6974    0.0100            0.7074
2.5      0.6908    0.0150            0.7058    ✅ 현재
3.0      0.6868    0.0200            0.7068
```

**문제점**:
- MSE는 2.5%만 개선 (0.7188 → 0.6908)
- 하지만 α가 1.0 → 2.5로 극단적 변화
- λ=0.01로는 penalty가 0.015밖에 안 됨 (MSE 개선 대비 너무 작음)

**더 강한 λ 적용 시**:
```
λ=0.05 적용:
Alpha    MSE       Penalty         Total      선택?
──────────────────────────────────────────────────
1.0      0.7188    0.0000         0.7188     ✅
1.5      0.7067    0.0250         0.7317
2.5      0.6908    0.0750         0.7658
```

→ **λ=0.05면 α=1.0 선택** (2.5% MSE 개선을 포기하고 안정성 확보)

## 💡 권장 λ 값 및 근거

### 📌 권장: λ = 0.03 ~ 0.05

#### Option A: λ = 0.03 (균형잡힌 접근)

**장점**:
- α 편차 0.5 이하: 대부분 허용 (실질적 개선 가능성 높음)
- α 편차 1.0 이상: 강하게 억제 (98.6% 악화 방지)
- Method별 차이 존중 (ACOS α=0.3 같은 경우 허용)

**예상 효과**:
```
현재 (λ=0.01):
  - α≥2.0: 42%
  - Degraded: 46.7%

예상 (λ=0.03):
  - α≥2.0: ~5-10% (대폭 감소)
  - Degraded: ~15-20% (절반 이하로 감소)
```

**계산 근거**:
```
MSE 개선 1% = 0.01 (절대값)
α 편차 1.0에 대한 penalty = 0.03

→ 1% MSE 개선을 얻으려면 α를 최대 0.33만큼만 이동
→ α 편차가 1.0이면 MSE를 3% 개선해야 정당화됨
→ 현실적으로 3% MSE 개선은 거의 불가능
→ α≥2.0 같은 극단적 선택 억제
```

#### Option B: λ = 0.05 (보수적 접근) ⭐ **추천**

**장점**:
- 더 강력한 regularization
- α 편차 0.5 이상은 상당한 MSE 개선이 있어야만 선택
- 극단적 α (≥2.0) 거의 완전 차단

**예상 효과**:
```
예상 (λ=0.05):
  - α≥2.0: <5% (거의 제거)
  - Degraded: ~10-15% (최소화)
  - Near baseline (0.9-1.1): ~60-70%
```

**계산 근거**:
```
α 편차 0.5에 대한 penalty = 0.025

→ 0.5 편차를 정당화하려면 MSE 2.5% 개선 필요
→ α 편차 1.0은 MSE 5% 개선 필요 (거의 불가능)
→ 실질적으로 α < 1.5 범위로 제한
```

#### Option C: λ = 0.02 (온건한 접근)

**장점**:
- 현재 λ=0.01보다 2배 강함
- 극단적 α는 억제하되 Method별 특성 존중

**단점**:
- 여전히 α≥2.0 일부 허용 가능성
- 46.7% → 30% 정도 degradation 감소 예상 (여전히 높음)

## 🎯 Method-Specific λ 전략 (Advanced)

현재 분석에서 Method마다 최적 α가 극단적으로 다름:
```
ACOS:      α=0.28 (성능 개선 +2.65%)
EUCLIDEAN: α=4.30 (성능 악화 -6.72%)
```

### Method별 맞춤 λ 전략

```python

    # α < 1.0 선호 (약한 penalty)
    'acos': 0.02,
    'ari': 0.02,
    'ami': 0.03,
    
    # α ≈ 1.0 (중간 penalty)
    'jaccard': 0.05,
    'cosine': 0.05,
    'itr': 0.05,
    'jmsd': 0.05,
    'msd': 0.05,
    'ipwr': 0.05,
    
    # α > 1.0 경향 (강한 penalty - 억제 필요!)
    'pcc': 0.10,
    'spcc': 0.10,
    'cpcc': 0.10,
    'euclidean': 0.15,
    'manhattan': 0.10,
    'chebyshev': 0.20,
    'src': 0.10,
    'kendall_tau_b': 0.10,
}

# 실험 코드에 적용
regularization_penalty = METHOD_SPECIFIC_LAMBDA.get(method, 0.05) * abs(alpha - 1.0)
```

**근거**:
- EUCLIDEAN, CHEBYSHEV: α≥4.0 선택 경향 → 매우 강한 억제 필요
- ACOS, ARI: α<1.0 선택 경향 & 실제 성능 개선 → 약한 억제
- 나머지: 균형잡힌 λ=0.05

## 🎓 이론적 배경

### Regularization의 본질

**L2 Regularization과의 유사성**:
```
Neural Network: Loss + λ × ||weights||²
Our case:       MSE  + λ × |α - 1.0|
```

**적절한 λ 선택 기준**:
```
λ = k × (typical improvement scale) / (typical deviation scale)

여기서:
- typical improvement scale = 1% MSE = 0.01
- typical deviation scale = 0.5 (α가 0.5 ~ 1.5 범위)

→ λ ≈ 0.01 / 0.5 = 0.02 (하한)
→ λ ≈ 0.05 / 0.5 = 0.10 (상한)

적정 범위: λ ∈ [0.02, 0.10]
권장: λ = 0.05 (중간값)
```

### Cross-Validation과 Regularization

**일반 원칙**:
```
Validation set이 작을수록 → 더 강한 regularization 필요

현재:
- Validation: 15% (상대적으로 작음)
- Test: 20%

→ Validation에서의 우연한 좋은 결과를 억제해야 함
→ λ=0.05 같은 강한 regularization 정당화
```

## ✅ 결론 및 최종 권장사항

### 🎯 최종 권장: λ = 0.05

**이유**:
1. **현재 λ=0.01은 명백히 너무 약함**
   - 42% α≥2.0, 46.7% degradation
   
2. **λ=0.05는 이론적/실증적으로 적절**
   - α 편차 0.5 → penalty 0.025 (2.5% MSE 개선 필요)
   - α 편차 1.0 → penalty 0.050 (5% MSE 개선 필요, 거의 불가능)
   - 극단적 α 효과적 억제
   
3. **예상 효과**
   - Degradation: 46.7% → ~12%
   - α≥2.0: 42% → <5%
   - 평균 성능: -1.21% → +0.5~1.0%
